3、你最拿得出手的科研项目（工作项目），能讲出细节吗，做了什么，遇到什么问题，怎么解决的，学到什么

我最近实现的 sql 函数批量执行。
在这里我深入研究了达梦数据库执行引擎的 sql 和 pl 之间的切换逻辑，发现目前在 sql 语句中调用自定义 sql 函数时候，存在大量的栈帧切换代价。比如一条 sql 语句中调用 sql 函数时，此时存在 sql 引擎和 pl 引擎之间的切换操作，需要新建一个函数栈帧，将数据压栈并传给 sql 函数，sql 函数执行后再将结果返回给 sql 引擎。我注意到栈帧频繁切换阶段会存在非常大的性能开销，想到可以使用批处理的方式将sql 函数的单词调用改为批量调用。

需要重构整个 sql 函数的调用逻辑，以及 sql 引擎和 pl 引擎之间的切换操作。

我对 sql 方法进行了改造，在解析阶段就判断当前 sql 方法是否满足批量执行的条件。对于满足条件的 sql 函数同时维护两套执行逻辑
同时调用阶段也根据调用条件来判断执行哪一套调用逻辑

学到了什么：
将开销最大的外层循环去除，循环下推到最核心的内层，可以极大的提高执行效率。
