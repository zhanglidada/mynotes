2.使用 vscode + cline + deepseek api 开发完成微信小程序后编译并调试：

`pnpm dev:mp-weixin` 进行热编译，将把 Vue3/TS 编译成微信小程序能看懂的 `wxml`, `wxss`, `js`。然后在`项目根目录/dist/dev/mp-weixin` 中才是小程序真正的根目录，也就是验证功能时，在小程序开发软件中打开的目录。


假设当前项目文件夹叫 `hardware-assistant`，目录结构如下：

```
hardware-assistant/  <-- 🛑 VS Code 打开的是这里 (源码根目录)
├── src/
├── package.json
├── vite.config.ts
└── dist/            <-- ⚡️ 编译产物在这里
    └── dev/
        └── mp-weixin/  <-- ✅ 微信开发者工具要指到这里！！！
            ├── app.json
            ├── project.config.json
            └── pages/
```


3.在命令行中使用`pnpm dev:mp-weixin` 热编译后，像启动 **MySQL**、**Redis** 或 **Tomcat**。启动后，光标会“卡”在那里（或者输出日志），进程一直挂起，`pnpm dev:mp-weixin` 就是一个服务。

看日志的最后一行：

```
DONE  Build complete. Watching for changes...
```

启动了一个文件系统监听器（File Watcher）。它现在正“盯着”你的 `src` 目录：

1. 当你修改了 `index.vue` 并按下 `Ctrl + S` 保存。
    
2. 这个进程会**瞬间捕获**到变动。
    
3. 它会**增量编译**修改的部分（毫秒级）。
    
4. 它会通知微信开发者工具进行**热更新**。
    

如果你发现它“退出”到了 Terminal（即出现了命令行提示符），那反而说明**服务崩了**，你需要重新运行它，否则你改代码后模拟器不会有任何反应。

6.小图标处理问题：

### 技术落地小贴士

在小程序 `pages.json` 中配置 `tabBar` 时，你需要注意以下几点：

1. **准备两套图**：
    
    - 每个 Tab 需要两张图：`iconPath` (未选中，通常是灰色/线性) 和 `selectedIconPath` (选中，通常是主题色/实心)。
        
2. **格式与大小**：
    
    - **格式**: 必须是 **PNG** (不支持 SVG 用于 Tabbar，虽然页面内支持)。
        
    - **背景**: 必须是**透明背景**。
        
    - **尺寸**: 建议 **81px × 81px** (这样在 2x 和 3x 屏幕上都很清晰)。
        
    - **文件大小**: 尽量小于 40kb。
        

**下一步建议：** 你可以先去 **Remix Icon** 找一套现成的“线性”和“实心”图标（最快），或者用上面的 Prompt 生成一套独特的图标。如果需要切图或去底，可以使用在线工具如 Remove.bg。





7.AI 智能体使用，以当前小程序举例：
### 协同作战：一个完整的开发流

**任务：开发“手机详情页”**

1. **第一步 (找 Steve)**：
    
    > "Steve，我们要开发手机详情页。请规划一下页面结构，手机用户最关心哪些参数（屏幕？电池？相机？）？请列出优先级。"
    
2. **第二步 (找 Woz)**：
    
    > "Woz，根据 Steve 的规划，请更新 `src/types/hardware.ts` 定义 Phone 的接口。然后写一个 Python 脚本去维基百科抓取 iPhone 15 的这些数据。"
    
3. **第三步 (找 Jony)**：
    
    > "Jony，Woz 已经把数据结构定好了。请按照 iOS 18 风格设计手机详情页。相机参数我想用一个独立的 Bento 卡片展示，屏幕参数用列表。请给代码。"
    
4. **第四步 (找 Woz)**：
    
    > "Woz，把 Jony 的 UI 代码和你的数据逻辑绑定起来，在 `onLoad` 时获取数据。"
    
5. **第五步 (找 Steve)**：
    
    > "Steve，页面做好了，你看看还有什么细节需要优化的？"
    

通过这种方式，你就能充分利用这三个智能体的特长，像指挥一个真实团队一样开发你的小程序了。
