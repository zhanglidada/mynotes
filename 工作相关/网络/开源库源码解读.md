# 二、网络库

## 1.1 libev

libev是一个事件循环处理框架，通过将各种信号分别封装为不同的事件，将IO事件，定时器，和信号统一起来，统一放在事件处理这一套框架下处理。

libev基于reactor模式实现，其内部主要包含ev_watcher，ev_loop 以及 ev_run三个主要角色。

整体使用流程如下：
初始化ev_loop，定义监视器和回调函数，监视器和回调函数关联，监视器和ev_loop关联，调用ev_run将ev_loop运行起来。

### 1.1.1 核心数据结构
1）监视器watcher

监视器包括一个基础监视器`ev_watcher`，链表节点监视器`ev_watcher_list`以及一个带时间的监视器`ev_watcher_time`

```
typedef struct ev_watcher
{
  EV_WATCHER (ev_watcher)
} ev_watcher;

typedef struct ev_watcher_list
{
  EV_WATCHER_LIST (ev_watcher_list)
} ev_watcher_list;

typedef struct ev_watcher_time
{
  EV_WATCHER_TIME (ev_watcher_time)
} ev_watcher_time;

// 所有 watcher 共享的成员
#define EV_WATCHER(type)           \
    int active;     /* 是否已注册到 loop */       \
    int pending;    /* 是否有待处理事件（索引+1） */ \
    EV_DECL_PRIORITY               \
    EV_COMMON       /* void *data */              \
    EV_CB_DECLARE(type)  /* 回调函数指针 */

// 链表式 watcher（IO、信号、子进程等——多个 watcher 可监控同一 fd/signal）
#define EV_WATCHER_LIST(type)      \
    EV_WATCHER(type)               \
    struct ev_watcher_list *next;

// 时间式 watcher（定时器、周期定时器）
#define EV_WATCHER_TIME(type)      \
    EV_WATCHER(type)               \
    ev_tstamp at;   /* 绝对超时时间 */

```




2）ANFD 

ANFD是一个文件描述符的状态表

```
typedef struct {
  WL head;                /* 监听该 fd 的 ev_io 链表头 */
  unsigned char events;   /* 当前已在后端注册的事件掩码 */
  unsigned char reify;    /* 脏标志：是否需要重新同步到后端 */
  unsigned char emask;    /* 后端专用标志（如 epoll 的 EPERM 标记） */
  unsigned char eflags;   /* 后端专用标志 */
#if EV_USE_EPOLL
  unsigned int egen;      /* 代际计数器，避免过时事件 */
#endif
} ANFD;
```

- **`head` 链表**：一个 fd 可以挂接多个 I/O watcher（如独立处理读/写），通过单向链表访问，插入/删除均为 O(1)。
    
- **`events` 与 `reify` 配合实现延迟批量同步**：应用层修改监听不直接调系统调用，只设 `reify` 标志并收藏到 `fdchanges` 数组。事件循环开始时，`fd_reify()` 遍历脏 fd，合并所有 watcher 的事件集，**仅调用一次** `epoll_ctl` 或等效接口更新内核状态。这极大降低了系统调用次数。
    
- **`egen` 代际计数器**：每次修改该 fd 的监听时递增。后端返回事件时将其与 fd 携带的 egen 对比，不同则丢弃。这彻底解决了 fd 关闭后立即被复用、旧事件仍然存留在内核队列导致的误触发问题，无需昂贵的 `epoll_ctl(DEL)` 清理。




2）ANPENDING 与优先级二维队列
```
typedef struct {
  W w;          /* 指向待回调的 watcher */
  int events;   /* 触发的事件掩码 */
} ANPENDING;

ANPENDING *pendings[NUMPRI];   /* 每个优先级的待处理事件数组 */
int pendingcnt[NUMPRI];        /* 每个优先级的事件数量 */
```

- **优先级隔离**：事件按照 watcher 自身的 `priority` 分别入列，高优先级的回调一定优先于低优先级执行，避免重要事件被饿死。
    
- **事件合并**：若一个 watcher 已经存在于 pending 队列中（`w->pending != 0`），再次 feed 事件时只合并 `events` 掩码，不重复插入。既节省内存也防止重复回调造成的逻辑错误。
    
- **数组存储**：每个优先级的 pending 项是连续内存的数组，遍历执行时具有极佳的 CPU 缓存局部性。


3）ANHE – 定时器堆元素

libev 使用 **4 叉最小堆** 管理定时器（和 periodic），堆
```
#if EV_HEAP_CACHE_AT
  typedef struct {
    ev_tstamp at;      /* 缓存 watcher 的超时时间 */
    WT w;              /* 指向 ev_timer / ev_periodic 的指针 */
  } ANHE;
  #define ANHE_at(he)  (he).at
  #define ANHE_w(he)   (he).w
#else
  typedef WT ANHE;     /* 若不缓存，直接存储 watcher 指针 */
#endif
```

- **O(1) 获取最早到期时间**：只需读取 `timers[HEAP0].at`，主循环即可快速计算阻塞超时 `waittime`，不需要遍历全部定时器。
    
- **O(log N) 插入/删除**：采用 4 叉堆（`DHEAP=4`）而非 2 叉堆，在 50000+ 定时器时约有 5% 的性能提升，因为每个节点比较 4 个子女能减少堆高度，改善缓存命中率。
    
- **缓存 `at`**：`ANHE_at_cache` 把 watcher 的 `at` 字段复制到堆元素中，使得调整堆时不需要解引用 watcher 指针，提升内存访问效率。
    
- **紧凑数组**：堆用数组存储，完全连续，遍历和重组（如时间跳跃后 `reheap`）均受益于空间局部性。

4）fdchanges – 延迟变更列表

```
int *fdchanges;    /* 动态数组，存储需要同步的 fd 号 */
int fdchangemax;   /* 数组容量 */
int fdchangecnt;   /* 当前待同步数量 */
```

- **批量投递内核**：`fd_reify` 一次性处理所有在 `fdchanges` 中的 fd，每个 fd 产生一次 `backend_modify`。相比每次 I/O 监听变化就立即系统调用，可削减一个数量级的内核交互。
    
- **自动去重**：`fd_change()` 中通过 `anfds[fd].reify` 标志避免同一个 fd 在数组中出现多次，即使短时间内对其反复修改，最终只在 `fd_reify` 中处理一次。
    
- **快照式处理**：先记录 `changecnt` 快照，然后在 `fd_reify` 过程中即使有新的 fd 变更（例如 io_uring 后端因队列满触发内联事件），也不会导致死循环或漏处理，仅将新增数据前移以备下次循环。




6）ev_loop 结构体 – 全局状态容器

struct ev_loop 包含所有后端状态、watcher 数组、时间变量等，支持多重循环独立实例。

```
struct ev_loop

{

ev_tstamp ev_rt_now;

#define ev_rt_now ((loop)->ev_rt_now)

#define VAR(name,decl) decl;

#include "ev_vars.h"

#undef VAR

};
```

- **数据局部性**：所有与一个事件循环相关的数据集中在同一结构体，避免全局变量和跨线程伪共享。
    
- **精准的时间变量**：`mn_now`（单调时钟）、`ev_rt_now`（实时时钟）、`rtmn_diff` 等缓存，避免在每次定时器检查时重复系统调用获取时间。
    
- **快速访问**：通过宏展开为 `loop->member` 直接访问，零额外函数调用开销。

7）辅助结构：ANFS（inotify 哈希）、ANHE 堆数组

`ANFS fs_hash[]`：用于 `ev_stat` 的 inotify 加速，以 watcher 描述符 `wd` 的哈希分散到多个链表，减少 inotify 事件分发时的遍历长度。
`ANHE timers[]` / `ANHE periodics[]`**：数组形式存储堆，紧凑连续，堆上浮/下沉操作缓存友好。



### 1.1.2 libev性能优点：

1. **延迟与批量化**：`ANFD.reify` + `fdchanges` → 极致的系统调用削减。
    
2. **最小堆**：`ANHE` + 4 叉堆 → O(log N) 定时器管理，O(1) 超时计算。
    
3. **优先级队列**：`pendings` 二维数组 + 事件合并 → 保证高优响应且避免重复回调。
    
4. **紧凑与连续存储**：数组堆、pending 数组、ANFD 数组→ 缓存命中率最大化。
    
5. **无冗余指针**：`ev_watcher_list` 仅一个 `next`，不用双向链表或动态哈希，节省内存并减少分配。
    
6. **防御式设计**：`egen` 代际计数器用极低成本消除 fd 重用导致的伪事件，避免了昂贵的清理操作。
